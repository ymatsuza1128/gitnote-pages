# お問い合わせ導線：メール → Google フォーム移行 設計

- 日付: 2026-06-19
- 状態: 承認済み（実装はフォーム URL 待ち）
- 対象リポジトリ: `ymatsuzatech-lp`（Jekyll / GitHub Pages、`main` から自動デプロイ）

## 背景・目的

現在、全アプリ LP の問い合わせ先は個人 Gmail (`ymatsuza.tech@gmail.com`) の `mailto:` リンク。
これを Google フォームに統一し、**サイト上から個人メールアドレスを完全に排除**する
（スパム/ボット収集の抑制、問い合わせの一元管理）。

## 決定事項

1. **方式**: Google フォームへ**リンク**（別タブで開く）。埋め込みや専用ページは作らない。
2. **範囲**: **全ページ完全置き換え**。フッター CTA・アプリ内のテスター/ベータ募集文・
   プライバシーポリシーの法的連絡先・プロフィール・FAQ、すべてのメール導線をフォームに。
3. **多言語**: **1 枚のバイリンガル・フォーム**（質問を日英併記）。JP/EN 両ページから同一 URL を参照。

## フォーム仕様（ユーザーが作成）

| 質問 | 形式 | 必須 |
|---|---|---|
| アプリ / App | プルダウン：GitMD / AIのたまご / DisplayMemory / きずなクリッカー / 感染拡大クリッカー / AI工場長 / その他・全般 | 任意 |
| ご用件 / Topic | プルダウン：一般質問 / 不具合報告 / テスター・ベータ希望 / プライバシー・データ削除 / その他 | 任意 |
| メール / Email（返信希望時のみ） | 記述式 | 任意 |
| 内容 / Message | 段落 | 必須 |

- 回答はスプレッドシートにリンク推奨。
- アプリ別の「事前入力 URL」を用意すれば、各アプリの募集文リンクにアプリ名/用件をプリセット可能（任意の強化）。

## 変更対象（約12ファイル・40箇所超の `mailto:`）

- フッター CTA: `index.html`, `gitmd.html`, `aiegg.html`, `displaymemory.html`,
  `outbreak-clicker.html`, `kizuna-clicker.html`, `ai-foreman/index.html`, `profile.html`
- アプリ内募集文（文言も「お問い合わせフォームよりご連絡ください」に調整）: 上記アプリページの先行ベータ/テスター案内
- プライバシーポリシー: `privacy-policy.html`, `aiegg-privacy.html`, `outbreak-clicker-privacy.html`
- FAQ: `faq.md`
- **残すもの**: `privacy-policy.html` / `faq.md` の「テスター向け Google グループ」リンクは有用なため維持。

## 進め方

1. ユーザーが上記仕様でフォームを作成 → フォーム URL を共有。
2. 実 URL で全ファイルを一括編集（ブランチ上で作業、プレースホルダのままデプロイしない）。
3. 差分を確認 → ユーザーの GO でコミット／デプロイ。

## 注意点

- プライバシーポリシーの連絡先をフォーム化するため、**フォームは削除せず維持**すること
  （消すと法的連絡先が切れる。メールより壊れやすい点に留意）。
- Google Play 審査・データ削除請求の窓口としてフォームリンクは一般に許容される。

## スコープ外

- フォーム本体の作成（ユーザーの Google アカウントで実施）。
- 埋め込み型フォームサービスの導入、お問い合わせ専用ページの新設。
